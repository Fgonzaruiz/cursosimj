01 — Herramientas

Claude, OpenAI y Gemini:
Qué hace cada una

Tres herramientas, un mismo objetivo: hacer tu trabajo más rápido y mejor. Aquí las diferencias clave para el equipo de SiMJ antes de empezar.

Claude

Anthropic · claude.ai

El modelo de Anthropic. Destaca en seguir instrucciones complejas con precisión, generar código estructurado, analizar documentos largos y mantener contexto. El más consistente siguiendo reglas de formato.

  • Código Laravel / PHP / SQL con contexto de repositorio
  • Análisis de documentos técnicos largos (hasta 200K tokens)
  • Revisión de PRs con criterios específicos
  • Redacción con restricciones estrictas de formato
  • Arquitectura y diseño de módulos de SiGestión
  • Tests unitarios y documentación técnica

OpenAI · ChatGPT

OpenAI · chatgpt.com

Los modelos de OpenAI. GPT-4o para el día a día creativo y de negocio. Los modelos o-series (o3, o4-mini) para razonamiento lógico complejo. Muy bueno para contenido, comunicación empresarial y búsqueda en tiempo real.

  • Propuestas comerciales y emails de cliente
  • Contenido para web, LinkedIn e Instagram de SiMJ
  • Resúmenes de reuniones y actas internas
  • Análisis de mercado con búsqueda web en tiempo real
  • Informes de actividad para clientes IT
  • Brainstorming de campañas y estrategia de producto

Gemini · Google

Google DeepMind · gemini.google.com

Los modelos de Google. Gemini 1.5 Pro/Flash para tareas multimodales (texto + imágenes + vídeo). Gemini 2.0 Pro para razonamiento avanzado. Nativo en Google Workspace (Docs, Sheets, Gmail) — muy útil para el equipo de admin de SiMJ.

  • Análisis de imágenes, capturas y documentos escaneados
  • Integración directa con Google Docs y Sheets
  • Búsqueda con grounding en tiempo real (Google Search)
  • Resúmenes de vídeos de formación (YouTube)
  • Procesamiento de PDFs e informes de clientes
  • Generación de código con contexto visual
💡
Regla práctica para SiMJ: ¿Código o análisis técnico profundo? → Claude. ¿Email, post o info actualizada? → ChatGPT. ¿Imagen, PDF escaneado o Google Workspace? → Gemini. Las tres se complementan.
02 — Fundamentos

Cómo construir
un buen prompt

Las reglas base son similares en las tres herramientas, pero hay diferencias importantes. Aquí las más críticas para el equipo de SiMJ.

Claude — Técnicas
Técnica 01 · Oficial Anthropic

XML Tags como Separadores Semánticos

Claude está entrenado para respetar XML tags como fronteras de significado. Separan contexto, código, instrucciones y tarea. La <tarea> va siempre al final.

Plantilla Base · Claude
<contexto> Proyecto: SiGestión ERP (Laravel 10, PHP 8.2, MySQL 8) Convenciones: PSR-12, sin comentarios innecesarios. </contexto> <codigo> [Pega aquí el código relevante] </codigo> <tarea> [Tu petición — siempre aquí, al final] </tarea>
Técnica 02 · +30% calidad

La Tarea Siempre al Final

Anthropic documenta hasta un 30% de mejora cuando, en prompts con código largo, la consulta va al final. El modelo "calienta" con el contexto antes de responder.

Orden Correcto
✅ CORRECTO: <contexto>Stack, convenciones...</contexto> <codigo>[500 líneas de código]</codigo> <tarea>Refactoriza para evitar N+1 queries.</tarea> ❌ MAL — tarea al inicio: "Refactoriza esta función:" + [500 líneas]
Técnica 03 · Anti-alucinación

Uncertainty Permission

Sin esta instrucción, Claude inventa antes de admitir que necesita más contexto. Crucial en código de SiGestión donde un error es costoso.

Fragmento Crítico
"Si necesitas ver más archivos del repositorio para responder correctamente, dímelo así: 'Necesito ver también: [archivo/clase]' No supongas cómo funciona ninguna clase custom de SiGestión sin haberla visto primero."
Técnica 04 · Opus 4.6

Extended Thinking para Arquitectura

Activa razonamiento profundo interno. El modelo piensa antes de responder. Usa budget_tokens: 8000–16000 para problemas de arquitectura.

Python · Anthropic API
import anthropic
client = anthropic.Anthropic()
response = client.messages.create(
  model="claude-opus-4-6",
  max_tokens=16000,
  thinking={
    "type": "enabled",
    "budget_tokens": 10000
  },
  messages=[{"role": "user",
    "content": "[Problema de arquitectura]"
  }]
)
Técnica 05 · Producción

Fijar Snapshot de Modelo en Código

Los aliases pueden apuntar a modelos distintos tras una actualización. En código de SiGestión: siempre el snapshot con fecha.

Producción vs Alias
✅ En código SiGestión: model = "claude-sonnet-4-6-20260217" ❌ Alias (puede cambiar sin aviso): model = "claude-sonnet-4-6"

OpenAI — Técnicas
Técnica 01 · Base

Contexto de Empresa Siempre Primero

ChatGPT no sabe a qué se dedica SiMJ. Sin contexto, produce respuestas genéricas. Un párrafo de contexto transforma completamente la calidad del output.

Plantilla Base · OpenAI
Somos SiMJ, empresa de software a medida y servicios IT en Córdoba. Desarrollamos SiGestión (ERP para PYMES), SiGPS (control de flotas) y ofrecemos ciberseguridad y mantenimiento IT. Nuestros clientes son PYMES y autónomos andaluces. Tono: cercano, profesional, sin tecnicismos. [Tu petición aquí]
Técnica 02 · Formato

Formato de Salida Explícito

Sin decirle qué quieres, ChatGPT elige. Especificar el formato convierte un resultado "usable" en un resultado "listo para enviar".

Comparativa
❌ Sin formato: "Escribe algo sobre nuestro módulo de facturación" ✅ Con formato: "Post LinkedIn sobre SiGestión: módulo facturación. 3 párrafos · máx 180 palabras Primera frase: engancha sin revelar todo Termina con pregunta al lector Máx 2 emojis · Sin hashtags genéricos"
Técnica 03 · Iteración

El Prompting es Conversación, No Magia

No busques el prompt perfecto al primer intento. ChatGPT mejora enormemente en conversación: genera → ajusta → refina.

Flujo de 3 Turnos
# Turno 1 — Borrador rápido "Redacta propuesta para cliente que quiere informatizar su taller de 5 personas." # Turno 2 — Ajuste con contexto SiMJ "Hazla más concreta: menciona SiGestión, máx 1 página, añade demo gratuita como CTA." # Turno 3 — Push de calidad "¿Qué le falta para que este cliente diga sí en vez de 'lo pensaré'?"
Técnica 04 · Análisis

Chain-of-Thought para Decisiones

Para tareas de análisis con GPT-4o: pide que razone antes de concluir. No usar con o3/o4-mini — ya razonan internamente.

Plantilla CoT
"Antes de responder, analiza brevemente: 1. ¿Qué necesita realmente este cliente? 2. ¿Qué obstáculo le impide decidir ahora? 3. ¿Qué solución de SiMJ encaja mejor? Luego responde a: [situación concreta]"
Técnica 05 · Datos Actuales

Búsqueda en Tiempo Real

Con la búsqueda web activada en ChatGPT, pide información actualizada. Ideal para benchmarks del sector, noticias de competidores o regulaciones recientes.

Con Búsqueda Web
"Busca las últimas noticias sobre [tema relevante para SiMJ — ley antifraude, ERP, ciberseguridad PYMES España]. Resume en 5 puntos clave para empresa IT que ofrece servicios a PYMES andaluzas. Incluye fecha de cada noticia. Solo fuentes de los últimos 60 días."

03 — Comparador Visual

Prompts reales:
el antes y el después

Los mismos objetivos, resueltos mal y bien. Cada par muestra exactamente qué falla y qué lo soluciona.

OpenAI · Propuesta comercial para cliente

✗ Así no
"Escríbeme una propuesta para un cliente que quiere software."
Por qué falla: Sin cliente, sin producto, sin tono. El resultado es un texto genérico que podría ser de cualquier empresa.
✓ Así sí
"Somos SiMJ (software a medida, Córdoba). El cliente es una clínica dental de 8 personas que lleva la agenda en papel y Excel. Quieren informatizarlo todo.

Propuesta de 1 página: el problema en su lenguaje, solución (SiGestión adaptado a clínicas), 3 beneficios concretos y CTA: demo gratuita. Tono cercano, sin tecnicismos."
Qué lo mejora: Cliente concreto, producto correcto, beneficios esperados, formato de 1 página, tono definido. Usable al primer intento.

Claude · Revisión de código Laravel

✗ Así no
"Revisa este código y dime qué mejorar."
[300 líneas de PHP]
Por qué falla: Sin foco, Claude revisa todo: estilo, rendimiento, seguridad. 25 comentarios sin saber cuál es urgente.
✓ Así sí
<contexto>Laravel 10, producción, 5.000 RPM. Controlador de facturación.</contexto>
<codigo>[código]</codigo>
<tarea>Revisa SOLO seguridad. Por cada problema: LÍNEA | QUÉ FALLA | FIX. Si no hay problemas: dilo sin rodeos.</tarea>
Qué lo mejora: Foco único, stack concreto, formato de salida, permiso para decir que todo está bien.
04 — Niveles

3 niveles de prompt
según la tarea

El nivel del prompt debe ser proporcional a la complejidad de la tarea. Más complejo = más contexto, más estructura, modelos más potentes.

Nivel 1 — BásicoTareas simples · Haiku 4.5 / GPT-4o-mini / Gemini Flash
💡
Prompts de 40-100 palabras, instrucción directa, sin estructura compleja. Ideal para tareas repetitivas del día a día.
Nivel 2 — MedioAnálisis con contexto · Sonnet 4.6 / GPT-4o / Gemini 1.5 Pro
💡
150-350 palabras. Claude: XML tags obligatorios. OpenAI: contexto empresa + restricciones + formato. Gemini: markdown con secciones claras.
Nivel 3 — AvanzadoEstrategia y arquitectura · Opus 4.6 / GPT-4o rol experto / Gemini 2.0 Pro
ℹ️
300-600 palabras. Diseño de módulos nuevos de SiGestión, decisiones de migración de infraestructura, estrategia de producto 2026. Claude con Extended Thinking; GPT-4o con rol experto; Gemini con grounding y razonamiento multi-paso.

06 — Privacidad

IA en empresa:
Qué puedes enviar y qué no

El mayor riesgo al usar IA en SiMJ no es técnico. Es enviar datos de clientes sin saber las implicaciones legales. Estas reglas aplican por igual a Claude, ChatGPT y Gemini.

⚠️
El riesgo real: Las IAs de consumo gratuitas pueden usar tus conversaciones para entrenar sus modelos. Enviar datos reales de clientes de SiMJ puede constituir una violación del RGPD (multas hasta el 4% del volumen global de negocio).
⛔ Nunca enviar a ninguna IA

Datos reales de clientes de SiMJ

Nombres, CIFs, emails, teléfonos, datos de contratos. Ni en screenshots, ni en copy-paste, ni como "ejemplo".

Credenciales y tokens de acceso

Claves API, passwords de servidores, .env files, tokens de GitHub. Aunque sea "solo para el contexto". Nunca.

Código con datos de producción

Queries con datos reales, logs con PII, dumps de base de datos. Anonimizar antes de pegar cualquier fragmento.

Estrategia interna confidencial

Márgenes, precios internos, negociaciones activas, decisiones de contratación. No son entornos seguros para esto.

✓ Puedes enviar sin problema

Descripciones genéricas de clientes

"Empresa de transporte con 12 vehículos" en lugar de nombres reales. Misma utilidad, cero riesgo legal.

Código sin credenciales

Funciones, clases, lógica de negocio de SiGestión. La lógica sin datos reales no es sensible.

Documentación y plantillas internas

Políticas sin PII, templates de propuestas, manuales de producto, documentación de API sin tokens.

Todo lo público de SiMJ

Textos de simj.es, posts de redes sociales, materiales de marketing ya publicados.

Con las herramientas de esta guía, un dev de SiMJ puede generar el 40-60% del código de scaffolding (migrations, factories, tests, docblocks) con IA en cuestión de segundos. El tiempo liberado va a lo que realmente aporta valor: lógica de negocio, revisión de calidad y relación con el cliente.
09 — Errores Comunes

Los 10 errores que
arruinan un prompt

Los más frecuentes en equipos que empiezan con IA. Haz clic en cada error para ver el fix.

🎭Crítico

El Experto Sin Credenciales

"Eres un experto en marketing." Sin empresa, sector, años. Respuestas de libro de texto.

Fórmula: [cargo] + [empresa/sector] + [sub-especialidad]. Para SiMJ: "Eres responsable de marketing de empresa de software a medida en Córdoba, especializado en captación de PYMES locales."
📐Crítico

Sin Formato de Salida

No especificar qué quieres: email, bullets, tabla, post. El modelo elige y raramente acierta.

Añade siempre: "Formato: [tipo] · Longitud: [límite] · Termina con: [CTA]". Cuanto más específico, más reproducible el resultado.
🌊Crítico

El Tsunami de Tareas

"Analiza, resume, traduce y crea un plan." 4 tareas = resultados mediocres en todas.

Máximo 2 tareas por prompt. Si necesitas más, encadénalas: el output del primero es el input del segundo.
🧢Crítico

Sin Permiso de Incertidumbre

El modelo inventa antes de admitir que necesita más contexto. Peligroso en código.

Añade: "Si el dato no está disponible, escribe: 'Dato no disponible — necesitaría: [X]'". Reduce alucinaciones hasta un 60%.
🤖Alto

CoT en o3/o4-mini

"Piensa paso a paso" con modelos razonadores. Ya razonan internamente — pedírselo empeora el resultado.

Con o3/o4-mini: prompt directo + contexto completo + tarea clara. CoT explícito solo con GPT-4o y Claude Sonnet sin extended thinking.
🔐Alto

Datos Reales en el Prompt

Credenciales, nombres de clientes o contratos sin anonimizar. Riesgo legal real.

30 segundos antes de pegar: nombres → [CLIENTE], credenciales → [REDACTED], IPs → [IP_PROD].
📊Alto

Alias de Modelo en Producción

Usar "claude-sonnet-4-6" en código. Puede cambiar sin aviso y romper el comportamiento esperado.

En código: snapshot con fecha: claude-sonnet-4-6-20260217. Comportamiento predecible indefinidamente.
🏗️Alto

El Prompt Monolito

Todo en un párrafo. El modelo no diferencia contexto, instrucciones y tarea.

Para Claude: XML tags. Para ChatGPT: saltos de línea y encabezados claros. Para Gemini: markdown con ## secciones. La estructura mejora la fidelidad del output.
🔄Medio

Solo Restricciones Negativas

"No uses jerga" sin decir qué sí usar. Las restricciones solas son ambiguas.

Cada "no" necesita su "sí": "No uses jerga técnica → usa lenguaje que entienda el gerente de una PYME sin conocimientos IT."
🧪Medio

El Prompt de 1 Ejemplo

Probar con un input y asumir que funciona. El 80% de los fallos aparecen con edge-cases.

Antes de usar en SiGestión: 10 test cases (3 normales, 3 edge, 2 ambiguos, 2 que rompan el formato). Usa el propio modelo para generarlos.
